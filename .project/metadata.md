# Article Metadata

Blog posts live in `src/content/posts/<post-id>/index.{ja,en}.md`.
Each Markdown file must start with TOML front matter delimited by `+++`.

## Required Fields

- `title`: non-empty string.
- `date`: `YYYY-MM-DD` or `"auto"`.

`date = "auto"` is resolved by `node scripts/generate_metadata.mjs` from the git
commit that first added the Markdown file. The generated date is written back to
the file.

## Optional Fields

- `tags`: array of strings. Empty arrays are valid.
- `auto_tags`: table. `count`, when present, must be a positive integer.
- `reading`: positive number used for the displayed reading time.
- `kana`: string used for Japanese sorting/search metadata.

`auto_tags` is processed by `node scripts/generate_metadata.mjs`. Manual tags are
kept, and the generator adds the requested number of non-duplicate tags before
removing `auto_tags`.

## AI Provider Key

The generator reads `GEMINI_API_KEY` from the environment when `auto_tags` or
`[sumup] mode = "auto"` must be resolved. The committed verification check does
not call the AI provider; it only fails when unresolved metadata remains. Store
local keys through 1Password and expose them through `op run`:

```sh
GEMINI_API_KEY=op://VAULT/ITEM/gemini_api_key
```

Keep the reference in `.op.env`; do not commit literal secret values. In GitHub
Actions, add `GEMINI_API_KEY` as an Actions secret only for workflows that
intentionally generate metadata.

## AI Prompt Files

Prompt files live under `src/content/prompts/` and are sent to Gemini as
`systemInstruction`:

- `src/content/prompts/tag-generation.md`
- `src/content/prompts/summary-generation.md`

Article title, locale, existing tags, known tags, and body are assembled by
`scripts/generate_metadata.mjs` as the user content for each request.

## Summary

Use `[sumup]`.

```toml
[sumup]
mode = "text"
text = "Short summary."
```

Valid modes:

- `text`: requires `text` and is emitted as `post.summary`.
- `none`: emits an empty summary.
- `auto`: generated by Gemini and written back as `mode = "text"` with `generated = true`.

## Thumbnail

Use `[thumbnail]`.

```toml
[thumbnail]
mode = "none"
```

Valid modes:

- `none`: is written back as `mode = "file"` with the default path from `src/content/metadata.toml` and `generated = true`.
- `file`: requires `path`.

The loader exposes the resolved value as `post.thumbnail`; rendering thumbnails
in the UI is a separate task.

## Validation

Run:

```sh
node scripts/check_frontmatter.mjs
```

To generate or refresh metadata:

```sh
npm run metadata:generate:op
```

To check that generated values are already written:

```sh
node scripts/generate_metadata.mjs --check
```

This check is static. It does not require `GEMINI_API_KEY`.

To inspect prompts without calling the API:

```sh
npm run metadata:preview-prompts
```

The full repository verification runs both the `frontmatter` and
`metadata_generate_check` phases from `.project/verification.toml`, followed by
metadata quality checks.
