# Article Metadata

Blog posts live in `src/content/posts/<post-id>/index.{ja,en}.md`.
Each Markdown file must start with TOML front matter delimited by `+++`.

## Required Fields

- `title`: non-empty string.
- `date`: `YYYY-MM-DD` or `"auto"`.

`date = "auto"` is resolved by `node scripts/generate_metadata.mjs` from the git
commit that first added the Markdown file. The generated date is written back to
the file.

## Optional Fields

- `tags`: array of strings. Empty arrays are valid.
- `auto_tags`: table. `count`, when present, must be a positive integer.
- `reading`: positive number used for the displayed reading time.
- `kana`: string used for Japanese sorting/search metadata.

`auto_tags` is processed by `node scripts/generate_metadata.mjs`. Manual tags are
kept, and the generator adds the requested number of non-duplicate tags before
removing `auto_tags`.

## AI Provider Keys

The generator reads two keys from the environment:

- `GEMINI_API_KEY` (free tier) when `auto_tags`, `[sumup] mode = "auto"`, or a
  thumbnail concept must be resolved.
- `OPENAI_API_KEY` (billing enabled) when `[thumbnail] mode = "auto"` must
  generate an image.

The committed verification check does not call any AI provider; it only fails
when unresolved metadata remains. Store local keys through 1Password and expose
them through `op run`:

```sh
GEMINI_API_KEY=op://VAULT/ITEM/gemini_api_key
OPENAI_API_KEY=op://VAULT/ITEM/openai_api_key
```

Keep the references in `.op.env`; do not commit literal secret values. In GitHub
Actions, add `GEMINI_API_KEY` and `OPENAI_API_KEY` as Actions secrets only for
workflows that intentionally generate metadata.

## Automation Workflow

`.github/workflows/generate-metadata.yml` runs `npm run metadata:generate` on
pushes to `main` that touch `src/content/posts/**`, `src/content/metadata.toml`,
or `src/content/prompts/**` (and on manual dispatch). It injects both Actions
secrets, runs `python3 scripts/verify.py --mode ci`, then commits any resolved
frontmatter and generated thumbnails with `[skip metadata][skip translate]` so it
does not loop or re-trigger the translation workflow.

## AI Prompt Files

Prompt files live under `src/content/prompts/`. The Gemini prompts are sent as
`systemInstruction`:

- `src/content/prompts/tag-generation.md`
- `src/content/prompts/summary-generation.md`
- `src/content/prompts/thumbnail-concept.md` (derives the image concept from the
  Japanese summary)

The image prompt is sent to OpenAI as the generation prompt:

- `src/content/prompts/thumbnail-generation.md` (contains the `{CONCEPT}`
  placeholder, replaced with the resolved concept)

Article title, locale, existing tags, known tags, and body are assembled by
`scripts/generate_metadata.mjs` as the user content for each text request.

## Summary

Use `[sumup]`.

```toml
[sumup]
mode = "text"
text = "Short summary."
```

Valid modes:

- `text`: requires `text` and is emitted as `post.summary`.
- `none`: emits an empty summary.
- `auto`: generated by Gemini and written back as `mode = "text"` with `generated = true`.

## Thumbnail

Use `[thumbnail]`.

```toml
[thumbnail]
mode = "auto"
```

Valid modes:

- `none`: is written back as `mode = "file"` with the default path from `src/content/metadata.toml` and `generated = true`.
- `file`: requires `path`.
- `auto`: generates a thumbnail image and is written back as `mode = "file"` with
  `path = "/thumbnails/<post-id>.png"` and `generated = true`.

New posts scaffolded by `npm run new:post` default to `mode = "auto"`.

### Auto thumbnail generation

`mode = "auto"` generates one image per post id, shared by both locales:

- The image is stored at `public/thumbnails/<post-id>.png` and referenced as
  `/thumbnails/<post-id>.png`. Generation is idempotent: if the file already
  exists it is reused instead of regenerated.
- The image subject (the `{CONCEPT}` placeholder in the image prompt) comes from
  an explicit `[thumbnail].concept` string when present, otherwise it is derived
  from the resolved Japanese `[sumup]` summary by Gemini. The Japanese summary is
  always the concept source so both locales share one deterministic image.
- The image is generated by OpenAI `gpt-image-2` configured under
  `[thumbnail_generation]` in `src/content/metadata.toml` (`model`, `size`,
  `quality`, prompt files). Image generation requires a billing-enabled key.

The loader exposes the resolved value as `post.thumbnail`; rendering thumbnails
in the UI is a separate task.

## Validation

Run:

```sh
node tests/check_frontmatter.mjs
```

To generate or refresh metadata:

```sh
npm run metadata:generate:op
```

To check that generated values are already written:

```sh
node scripts/generate_metadata.mjs --check
```

This check is static. It does not require `GEMINI_API_KEY`.

To inspect prompts without calling the API:

```sh
npm run metadata:preview-prompts
```

The full repository verification runs both the `frontmatter` and
`metadata_generate_check` phases from `.project/verification.toml`, followed by
metadata quality checks.
